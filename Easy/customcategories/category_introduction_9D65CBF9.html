Main Description

Easy Introduction

Easy process é um framework Ágil de gestão de projetos usado para entregar aos clientes, de forma iterativa e incremental software de alto valor. Easy process depende de equipes hábeis e auto-organizadas para entregar os incrementos do software. Também depende de um cliente, ou Dono do Produto, que indique uma equipe com uma lista de funcionalidades desejadas, e que use o valor de negocio como mecanismo de priorização.

No easy processos projetos acontecem em uma série de iterações, com um uma semana de duração, chamadas incrementos (sprints).

Easy process é ideal para projetos cujos requisitos mudam rapidamente ou são altamente emergentes. O trabalho é registrado nas Pendências do software (Easy Backlog), que é uma lista de todos os desejos de mudança no sistema. No inicio de cada incremento é feita uma Reunião de Planejamento de Incremento (Sprint Planning) na qual o Dono do Produto (Easy Owner) prioriza as Pendências do software (easy Backlog), e a Equipe de desenvolvimento (Easy Team) seleciona as tarefas que ela pode completar durante o próximo Incremento. Essas tarefas são então movidas das Pendências do software para as Pendências do Incremento a serem desenvolvidas naquela sprint.

Durante um incremento, são conduzidas curtas reuniões diárias chamadas de Easy Diário (Daily Easy), que ajudam a equipe a manter-se no rumo. Também é registrado em um documento o que o membro da equipe esta fazendo naquele dia. Por exemplo, o desenvolvedor tem 3 tarefas para realizar durante 1 sprint, a medida que ele for finalizando e entrando em outra tarefa deve ser registrado no documento (uma planilha) essa mudança. Ela será usada para realização das reuniões de fim da sprint para avaliar o desempenho do membro para com a equipe.

Ao final de cada incremento a equipe demonstra a funcionalidade concluída, na Reunião de Revisão do Incremento (Sprint Review). Também é realizado a retrospectiva da sprint, onde de posse do documento citado no parágrafo anterior, a equipe vai se avaliar apontando pontos positivos e negativos para melhorar nas próximas iterações do projeto.


Roles

Existem três papéis :

  • O Product Owner (ou customer) Possui o produto e prioriza a lista de recursos desejados, também exerce função de gerencia no projeto.
  • O multifuncional Team Organize-se para construir o produto.
  • O Stakeholder é a parte interessada no produto.

Artefato

Easy define três artefatos com os quais as interações interagem durante a vida do projeto:

  • O Product Backlog é uma lista priorizada de recursos desejados.
  • O Sprint Backlog é uma lista prioritária de tarefas que a equipe identificou para o Sprint atual (iteração).
  • O Increment of Product Functionality  que é entregue no final de cada Sprint.

Encontros 

Há quatro reuniões, ou pontos de colaboração, que são usados ​​durante cada Sprint:

  • Sprint Planning
  • Daily Easy
  • Revisao da Sprint
  • Retrospectiva da Sprint
